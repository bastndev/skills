## 三个阶段

| 阶段 | 目的 | 主要能力 | 技能示例 | 状态 |
| ---------- | ------------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------- | ------ |
| **Start** | 启动项目 — 创建新项目或从现有项目学习 | 从第一个 commit 开始创建生产就绪的结构和配置。或分析现有代码库的架构和包并进行文档化，以便在自己的项目中复用有效的内容。 | `start-package`, `start-astro`, `skrapi` | ㅤㅤ✅ |
| **Middle** | 持续改进与打磨 | 评估项目（0–100 健康概览），然后**一次改进一个维度** — 性能、UI/UX、整洁度（文件顺序 + 注释规范）、安全性、结构、清理或代码质量 — 使用集中计划，仅在输入 `go` 时执行。 | `middle` | ㅤㅤ✅ |
| **End** | 审计、诊断与安全重构 | 全面的架构和质量分析。分类的发现结果，附带文件级证据。优先级排序的分阶段计划，**仅在明确批准后执行**。行为保持。多运行时支持。 | `end` | ㅤㅤ✅ |

按照你自然需要的顺序排列 — **Start** 新项目（或研究现有代码库获取灵感），在 **Middle** 中优化，在 **End** 中加固。

| **[skrapi](./skills/skrapi/README.md)** | _Start_ — 指向你欣赏的项目，它会将该代码库的构建方式映射到固定 `SKRAPI/` 文件夹中的专注 Markdown — 架构、依赖和粘贴就绪的提示 — 以便在开始前借用适合你自己项目的模式。适用于任何技术栈（Web、移动端、扩展、库、monorepo）；每个描述都基于实际代码验证，而非从包名猜测。多语言输出（🇪🇸 ES · 🇺🇸 EN · 🇨🇳 ZH）。<br><br>→ [完整文档](./skills/skrapi/README.md) |

## 安装

每个技能的安装方式相同 — 选择你需要的：

```bash
npx skills add bastndev/skills --skill start-package   # Start  — scaffold a TS npm package
npx skills add bastndev/skills --skill start-astro     # Start  — scaffold an Astro project
npx skills add bastndev/skills --skill skrapi          # Start  — study & document any codebase
npx skills add bastndev/skills --skill middle          # Middle — score & improve one dimension
npx skills add bastndev/skills --skill end             # End    — audit & safely refactor
npx skills add bastndev/skills --skill l10n-sync       # Utility — sync localized assets safely
```

## End 技能的工作原理

`end` 是套件中最成熟的技能。这是它从头到尾的工作流程：

1. **首先分析** — 映射入口点并理解项目。**不修改任何文件。**
2. **结构化报告** — 清晰的发现结果分为 Bug（按严重程度）、债务/风险和建议，加上评分的健康概览、架构建议和有序计划 — 所有都由具体的文件 + 行引用支持。
3. **你授权每个阶段** — 一次**恰好执行一个阶段**。每个阶段后，你会收到更改的精确摘要、执行的验证和剩余阶段列表。
4. **完全控制与安全** — 如果项目原本没有测试，从不创建。从不未经许可添加依赖或更改包管理器。尊重你未提交的工作，除非修复有理由的 bug，始终保留当前行为。

要获取完整工作流程、精确报告格式（包括所需的结束块）、架构决策规则和所有安全保证，请阅读专用技能文档：

→ **[End – Refactor Project](./skills/end/README.md)**

完整的内部规范位于 [skills/end/SKILL.md](./skills/end/SKILL.md)。

## 路线图

- **Start** — `start-package`（脚手架）、`start-astro`（Astro 脚手架）和 `skrapi`（研究现有代码库）今天可用；更多 `start-*` 脚手架正在开发中。
- **Middle** — `middle` 今天可用（健康概览 + 六个编号的焦点改进器）；按维度的更深入工具正在规划中。
- **End** — `end` 今天可用；更多运行时、额外的专门重构模式和实用工具正在规划中。
- **Utility** — `l10n-sync` 今天可用，用于纪律严明、令牌高效的工作空间本地化。
