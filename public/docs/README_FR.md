## Les trois phases

| Phase | Objectif | Capacités clés | Exemples de compétences | Statut |
| --- | --- | --- | --- | --- |
| **Start** | Initialisation du projet | Créer des structures de dossiers prêtes pour la production, initialiser les frameworks, appliquer les meilleures pratiques dès le premier commit. | `start-nextjs`, `start-vite`... | Prévu |
| **Middle** | Amélioration continue | Améliorer l'UI/UX, renforcer la sécurité, augmenter les performances, nettoyer la logique, éliminer le code mort. | Optimiseurs (TBD) | Prévu |
| **End** | Audit & Refactorisation | Analyse complète de l'architecture. Plan par étapes priorisé et exécuté **uniquement avec approbation**. Préserve le comportement. | `refactor-project` | ✅ |

## Installation

```bash
# Ajouter la compétence actuelle (End / refactor-project)
npx skills add bastndev/skills --skill end
```

Les futures compétences seront installables de la même manière:

```bash
npx skills add bastndev/skills --skill start-nextjs
```

## Comment fonctionne la compétence End

1. **Analyse uniquement** — Cartographie les points d'entrée, comprend la structure. **Zéro modification.**
2. **Rapport structuré** — Les résultats sont séparés en Bugs, Risques, Opportunités et Dette technique.
3. **Plan sur mesure** — Les phases sont ordonnées en fonction de la valeur réelle pour votre projet.
4. **Exécution étape par étape** — Exécute _une_ phase après votre autorisation et attend.
5. **La sécurité avant tout** — Ne crée jamais de tests s'il n'y en avait pas. N'ajoute pas de dépendances sans autorisation.

Voir la spécification complète dans [skills/end/SKILL.md](../../skills/end/SKILL.md).

## Feuille de route

- Famille **Start**: Générateurs de projets en une commande pour les piles les plus populaires.
- Famille **Middle**: Spécialistes en améliorations ciblées.
