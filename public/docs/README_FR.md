## Les trois phases

| Phase | Objectif | Capacités clés | Exemples de compétences | Statut |
| --- | --- | --- | --- | --- |
| **Start** | Initialisation du projet | Créer des structures de dossiers prêtes pour la production, initialiser les frameworks, appliquer les meilleures pratiques dès le premier commit. | `start-nextjs`, `start-vite`... | Prévu |
| **Middle** | Amélioration continue | Améliorer l'UI/UX, renforcer la sécurité, augmenter les performances, nettoyer la logique, éliminer le code mort. | Optimiseurs (TBD) | Prévu |

## Comment fonctionne le Skill End

1. **Analyse d'abord** — Cartographie les points d'entrée et comprend le projet. **Aucun fichier n'est modifié.**
2. **Rapport structuré** — Des résultats clairs dans quatre catégories (Bugs Confirmés avec gravité, Risques, Opportunités de Refactoring, Dette Technique) + une recommandation d'architecture et un plan ordonné. Tous les éléments incluent des références concrètes de fichiers et de lignes.
3. **Vous autorisez chaque phase** — Il exécute **exactement une phase** à la fois. Après chaque phase, vous obtenez un résumé précis des modifications, des validations effectuées et la liste des phases restantes.
4. **Contrôle total et sécurité** — Ne crée jamais de tests si le projet n'en avait aucun. N'ajoute jamais de dépendances ou ne modifie le gestionnaire de paquets sans permission. Respecte votre travail non commité et préserve toujours le comportement actuel, sauf s'il corrige un bug justifié.

Pour le workflow complet, les formats exacts des rapports, les règles de décision d'architecture et toutes les garanties de sécurité, lisez la documentation dédiée du skill :

→ **[End – Refactor Project](../../skills/end/README.md)**

## Feuille de route

- **Skills Start** — Scaffolding de projets en une commande pour les stacks populaires (Next.js, Vite, FastAPI, etc.)
- **Skills Middle** — Améliorateurs ciblés à la demande (performance, sécurité, UX, suppression de code mort, etc.)
- **Extensions End** — Plus de runtimes, modes de refactoring spécialisés supplémentaires et utilitaires.

Chaque skill aura sa propre documentation dédiée (comme l'actuel [End – Refactor Project](../../skills/end/README.md)).
