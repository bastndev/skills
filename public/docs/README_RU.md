## Три фазы

| Фаза | Цель | Ключевые возможности | Примеры навыков | Статус |
| ---------- | ------------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------- | ------ |
| **Start** | Начать проект — создать новый или изучить существующий | Создавайте структуры и конфигурации, готовые к продакшн, с первого коммита. Или анализируйте архитектуру и пакеты существующего кодовой базы и документируйте, чтобы переиспользовать то, что работает, в собственном проекте. | `start-package`, `start-astro`, `skrapi` | ㅤㅤ✅ |
| **Middle** | Непрерывное улучшение и шлифовка | Оцените проект (обзор здоровья 0–100), затем улучшайте **по одному измерению за раз** — производительность, UI/UX, порядок (порядок файлов + гигиена комментариев), безопасность, структура, очистка или качество кода — с целевым планом, выполняемым только при вводе `go`. | `middle` | ㅤㅤ✅ |
| **End** | Аудит, диагностика и безопасная рефакторинг | Полный анализ архитектуры и качества. Классифицированные результаты с доказательствами на уровне файлов. Приоритизированный пофазный план, выполняемый **только с явным одобрением**. Сохранение поведения. Поддержка нескольких сред выполнения. | `end` | ㅤㅤ✅ |

## Доступные навыки

Перечислены в естественном порядке, в котором вы к ним обратитесь — **Начните** что-то новое (или изучите существующий кодовую базу для вдохновения), улучшите в **Середине** и укрепите в **Конце**.

| Навык | Описание |
| ----------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **[start-package](./skills/start-package/README.md)** | _Start_ — Создаёт публикуемый TypeScript-пакет **двойного ESM + CJS** с bundled-объявлениями типов, сборкой `tsup` без конфигурации и TypeScript, зафиксированным на `5.x`, чтобы он чисто собирался как в CLI, так и в редакторе. Генерирует `package.json`, `tsconfig.json`, конфигурацию tsup, тест на работоспособность и настройки `.vscode`, затем устанавливает, собирает и проверяет.<br><br>→ [Полная документация](./skills/start-package/README.md) |
| **[start-astro](./skills/start-astro/README.md)** | _Start_ — Создаёт новый проект Astro, используя шаблон `minimal`, с чистой, **масштабируемой** архитектурой — готовой расти от портфолио до полноценного приложения. Настраивает общий макет, шапку, подвал, страницы, переключение темы светлой/тёмной, нативные View Transitions, псевдоним пути и Content Collections из коробки.<br><br>→ [Полная документация](./skills/start-astro/README.md) |
| **[skrapi](./skills/skrapi/README.md)** | _Start_ — Направьте его на проект, который вы восхищаетесь, и он покажет, как этот кодовая база построена в фиксированную папку `SKRAPI/` из целевого Markdown — архитектура, зависимости и промпты готовые к копированию — чтобы вы могли заимствовать паттерны, которые подходят вашему проекту, до начала работы. Работает на любом стеке (веб, мобильные, расширения, библиотеки, монорепо); каждое описание проверяется по реальному коду, никогда не угадывается по имени пакета. Многоязычный вывод (🇪🇸 ES · 🇺🇸 EN · 🇨🇳 ZH).<br><br>→ [Полная документация](./skills/skrapi/README.md) |
| **[middle](./skills/middle/README.md)** | _Middle_ — Нумерованные улучшатели по запросу для активной разработки. `0` оценивает ваш проект или папку с обзором здоровья 0–100 и указывает на самую слабую область; `1–7` квалифицирует каждое одно измерение (⚡ производительность · 🎨 ui-ux · 🗂️ порядок · 🔒 безопасность · 🏗️ структура · 🧹 очистка · 🧩 качество), сообщает выводы, подтверждённые доказательствами, и предлагает план коррекции — выполняемый пофазно только при вводе `go`.<br><br>→ [Полная документация](./skills/middle/README.md) |
| **[end](./skills/end/README.md)** | _End_ — Понимает ваш проект от начала до конца. Предоставляет чёткую диагностику (подтверждённые баги, риски, возможности, технический долг) с конкретными ссылками, рекомендует правильное архитектурное направление для _этой_ кодовой базы и строит упорядоченный план выполнения. Каждое изменение происходит в изолированной, отзываемой фазе — выполняется только при вводе `go`, `start` или `proceed`, и никакие файлы не затрагиваются во время анализа.<br><br>→ [Полная документация и примеры](./skills/end/README.md) |
| **[l10n-sync](./skills/l10n-sync/README.md)** | _Utility_ — Дисциплинированный, экономящий токены навык локализации. Синхронизирует переведённые ресурсы с английским источником правды без полного перевода файла. Гарантирует сохранение инвариантов (код, HTML) и точных структур на 11 языках.<br><br>→ [Полная документация](./skills/l10n-sync/README.md) |

## Установка

Каждый навык устанавливается одинаково — выберите тот, который вам нужен:

```bash
npx skills add bastndev/skills --skill start-package   # Start  — scaffold a TS npm package
npx skills add bastndev/skills --skill start-astro     # Start  — scaffold an Astro project
npx skills add bastndev/skills --skill skrapi          # Start  — study & document any codebase
npx skills add bastndev/skills --skill middle          # Middle — score & improve one dimension
npx skills add bastndev/skills --skill end             # End    — audit & safely refactor
npx skills add bastndev/skills --skill l10n-sync       # Utility — sync localized assets safely
```

## Как работает навык End

`end` — самый зрелый навык в наборе. Вот его рабочий процесс от начала до конца:

1. **Сначала анализ** — Определяет точки входа и понимает проект. **Файлы не изменяются.**
2. **Структурированный отчёт** — Чёткие выводы, классифицированные на Баги (с серьёзностью), Долг/Риски и Предложения, плюс оценённый обзор здоровья, рекомендация по архитектуре и упорядоченный план — всё подтверждено конкретными ссылками на файл + строку.
3. **Вы авторизуете каждую фазу** — Выполняет **ровно одну фазу** за раз. После каждой фазы вы получаете точное резюме изменений, выполненных проверок и список оставшихся фаз.
4. **Полный контроль и безопасность** — Никогда не создаёт тесты, если их не было в проекте. Никогда не добавляет зависимости и не меняет менеджер пакетов без разрешения. Уважает вашу незакоммиченную работу и всегда сохраняет текущее поведение, кроме исправления обоснованного бага.

Для полного рабочего процесса, точных форматов отчётов (включая обязательные закрывающие блоки), правил архитектурных решений и всех гарантий безопасности прочитайте специальную документацию навыка:

→ **[End – Refactor Project](./skills/end/README.md)**

Полная внутренняя спецификация находится в [skills/end/SKILL.md](./skills/end/SKILL.md).

## Дорожная карта

- **Start** — `start-package` (каркас), `start-astro` (каркас Astro) и `skrapi` (изучение существующей кодовой базы) доступны сегодня; больше генераторов `start-*` в разработке.
- **Middle** — `middle` доступна сегодня (обзор здоровья + шесть нумерованных улучшателей); более глубокие инструменты по измерениям запланированы.
- **End** — `end` доступна сегодня; дополнительные среды выполнения, специализированные режимы рефакторинга и утилиты запланированы.
- **Utility** — `l10n-sync` доступна сегодня для дисциплинированной, экономящей токены локализации рабочего пространства.

Каждый навык поставляется со своей собственной документацией (как текущий [End – Refactor Project](./skills/end/README.md)).
