## Три Фазы

| Фаза | Назначение | Ключевые возможности | Примеры навыков | Статус |
| ---------- | ------------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------- | ------ |
| **Start** (Начало) | Начать проект — создать новый или изучить существующий | Создавайте готовые к продакшену структуры и конфигурации начиная с коммита #1. Или проанализируйте архитектуру и пакеты существующей кодовой базы и задокументируйте их, чтобы использовать рабочие паттерны в собственном проекте. | `start-package`, `start-astro`, `skrapi`       | ㅤㅤ✅ |
| **Middle** (Середина) | Непрерывное улучшение и полировка | Улучшайте UI/UX, усиливайте безопасность, повышайте производительность, очищайте логику и удаляйте мертвый код во время активной разработки. | Целевые улучшения (определяется) | Запланировано |
| **End** (Конец) | Аудит, диагностика и безопасный рефакторинг | Полный анализ архитектуры и качества. Категоризированные результаты с доказательствами на уровне файлов. Приоритетный поэтапный план, выполняемый **только с явного разрешения**. Сохранение поведения. Поддержка множественных сред выполнения. | `end` | ㅤㅤ✅ |

## Доступные навыки

Перечислены в естественном порядке использования — **Start** (Начните) что-то новое (или изучите существующую кодовую базу для вдохновения), доработайте в **Middle** (Середине) и укрепите в **End** (Конце).

| Навык | Описание |
| ----------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **[start-package](../../skills/start-package/README.md)** | _Start_ (Начало) — Создает публикуемый TypeScript-пакет **dual ESM + CJS** со встроенными объявлениями типов, сборкой `tsup` без конфигурации и версией TypeScript, зафиксированной на `5.x`, чтобы он чисто собирался как в CLI, так и в редакторе. Генерирует `package.json`, `tsconfig.json`, конфигурацию tsup, smoke-тест и настройки `.vscode`, а затем устанавливает, собирает и проверяет.<br><br>→ [Полная документация](../../skills/start-package/README.md) |
| **[start-astro](../../skills/start-astro/README.md)** | _Start_ (Начало) — Создает новый проект Astro, используя шаблон `minimal`, поверх которого наложена чистая и **масштабируемая** архитектура — готовая вырасти из портфолио в полноценное приложение. Из коробки настраивает общий макет, заголовок, подвал, страницы, переключатель светлой/темной темы, нативные View Transitions, псевдонимы путей и коллекции контента (Content Collections).<br><br>→ [Полная документация](../../skills/start-astro/README.md) |
| **[skrapi](../../skills/skrapi/README.md)**               | _Start_ (Начало) — Укажите на проект, которым вы восхищаетесь, и навык отобразит, как построена эта кодовая база, в фиксированной папке `SKRAPI/` с целенаправленным Markdown — архитектура, зависимости и готовые к вставке промпты — чтобы вы могли позаимствовать подходящие шаблоны для своего собственного проекта перед началом работы. Работает на любом стеке (веб, мобильные, расширения, библиотеки, монорепозитории); каждое описание проверяется на реальном коде, никогда не угадывается по имени пакета. Многоязычный вывод (🇪🇸 ES · 🇺🇸 EN · 🇨🇳 ZH).<br><br>→ [Полная документация](../../skills/skrapi/README.md) |
| **[end](../../skills/end/README.md)**                     | _End_ (Конец) — Понимает ваш проект от начала до конца. Предоставляет четкий диагноз (подтвержденные баги, риски, возможности, технический долг) с конкретными ссылками, рекомендует правильное архитектурное направление для _этой_ кодовой базы и строит упорядоченный план выполнения. Каждое изменение происходит в изолированной проверяемой фазе — оно продолжается только тогда, когда вы скажете `go`, `start` или `proceed`, и ни один файл не изменяется во время анализа.<br><br>→ [Полная документация и примеры](../../skills/end/README.md) |

> **Примечание:** Каждый навык поставляется со своим подробным README. На главной странице дается общий обзор; загляните в `./skills/<skill>/README.md` для углубленного использования, примеров отчетов и гарантий.

## Установка

Каждый навык устанавливается одинаково — выберите тот, который вам нужен:

```bash
npx skills add bastndev/skills --skill start-package   # Start (Начало) — создать структуру npm пакета TS
npx skills add bastndev/skills --skill start-astro     # Start (Начало) — создание проекта Astro
npx skills add bastndev/skills --skill skrapi          # Start (Начало) — изучить и задокументировать любую кодовую базу
npx skills add bastndev/skills --skill end             # End (Конец) — провести аудит и безопасный рефакторинг
```

## Как работает навык "End"

`end` — самый зрелый навык в наборе. Вот его сквозной рабочий процесс:

1. **Сначала анализ** — Картирует точки входа и понимает проект. **Ни один файл не изменяется.**
2. **Структурированный отчет** — Четкие выводы, разбитые на категории: Баги (с серьезностью), Долг/Риски и Предложения, плюс оцененный обзор работоспособности, архитектурная рекомендация и упорядоченный план — все это подкреплено конкретными ссылками на файлы и строки.
3. **Вы разрешаете каждую фазу** — Он выполняет **ровно одну фазу** за раз. После каждой фазы вы получаете точное резюме изменений, выполненных проверок и список оставшихся фаз.
4. **Полный контроль и безопасность** — Никогда не создает тесты, если в проекте их не было. Никогда не добавляет зависимости или не изменяет менеджер пакетов без разрешения. Уважает вашу незафиксированную работу (uncommitted) и всегда сохраняет текущее поведение, если только не исправляет обоснованный баг.

Чтобы ознакомиться с полным рабочим процессом, точными форматами отчетов (включая обязательные закрывающие блоки), правилами принятия архитектурных решений и всеми гарантиями безопасности, прочтите специальную документацию по навыку:

→ **[End – Рефакторинг проекта](../../skills/end/README.md)**

Полная внутренняя спецификация находится в [skills/end/SKILL.md](../../skills/end/SKILL.md).

## Дорожная карта

- **Start** (Начало) — `start-package` (создание структуры), `start-astro` (создание Astro) и `skrapi` (изучение существующей кодовой базы) доступны уже сегодня; на подходе новые `start-*` генераторы структур.
- **Middle** (Середина) — запланированы целенаправленные инструменты улучшения по требованию (производительность, безопасность, UX, удаление мертвого кода).
- **End** (Конец) — `end` доступен уже сегодня; запланировано больше сред выполнения, дополнительные специализированные режимы рефакторинга и утилиты.

Каждый навык поставляется со своей собственной выделенной документацией (как текущий [End – Рефакторинг проекта](../../skills/end/README.md)).
