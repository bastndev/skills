## उपलब्ध स्किल्स

उन्हें उस प्राकृतिक क्रम में सूचीबद्ध किया गया है जिसमें आप उनका उपयोग करेंगे — कुछ नया **Start** (शुरू) करें (या प्रेरणा के लिए मौजूदा कोडबेस का अध्ययन करें), इसे **Middle** (मध्य) में सुधारें, और इसे **End** (अंत) में मजबूत करें।

## इंस्टॉलेशन

प्रत्येक स्किल उसी तरह से इंस्टॉल की जाती है — जो आपको चाहिए उसे चुनें:

```bash
npx skills add bastndev/skills --skill start-package   # Start (शुरुआत) — एक TS npm पैकेज का ढांचा बनाएं
npx skills add bastndev/skills --skill start-astro     # Start (शुरू करें) — Astro प्रोजेक्ट बनाएं
npx skills add bastndev/skills --skill skrapi          # Start (शुरुआत) — किसी भी कोडबेस का अध्ययन और दस्तावेज़ीकरण करें
npx skills add bastndev/skills --skill end             # End (अंत) — ऑडिट और सुरक्षित रूप से रिफैक्टरींग करें
```

## "End" स्किल कैसे काम करती है

`end` इस सूट में सबसे परिपक्व (mature) स्किल है। यहाँ इसका शुरू से अंत तक वर्कफ़्लो है:

1. **पहले विश्लेषण** — एंट्री पॉइंट्स को मैप करता है और प्रोजेक्ट को समझता है। **शून्य फ़ाइलें संशोधित की जाती हैं।**
2. **संरचित (Structured) रिपोर्ट** — बग्स (गंभीरता के साथ), ऋण/जोखिम और सुझावों में वर्गीकृत स्पष्ट निष्कर्ष, साथ ही स्कोर किया हुआ स्वास्थ्य अवलोकन, आर्किटेक्चर सिफारिश और एक क्रमबद्ध योजना — यह सब ठोस फ़ाइल + लाइन संदर्भों द्वारा समर्थित है।
3. **आप प्रत्येक चरण को अधिकृत करते हैं** — यह एक समय में **ठीक एक चरण** निष्पादित करता है। प्रत्येक चरण के बाद आपको परिवर्तनों का सटीक सारांश, किए गए सत्यापन और शेष चरणों की सूची मिलती है।
4. **पूर्ण नियंत्रण और सुरक्षा** — अगर प्रोजेक्ट में कोई परीक्षण (tests) नहीं था तो कभी परीक्षण नहीं बनाता है। बिना अनुमति के कभी भी निर्भरताएँ (dependencies) नहीं जोड़ता या पैकेज मैनेजर को नहीं बदलता है। आपके अनकमिटेड कार्य का सम्मान करता है और हमेशा वर्तमान व्यवहार को संरक्षित करता है जब तक कि किसी उचित बग को ठीक न कर रहा हो।

पूर्ण वर्कफ़्लो, सटीक रिपोर्ट फ़ॉर्मैट (आवश्यक समापन ब्लॉक सहित), आर्किटेक्चर निर्णय नियम और सभी सुरक्षा गारंटी जानने के लिए, समर्पित स्किल दस्तावेज़ पढ़ें:

→ **[End – प्रोजेक्ट को रिफैक्टरींग करें](../../skills/end/README.md)**

पूर्ण आंतरिक विनिर्देश (specification) [skills/end/SKILL.md](../../skills/end/SKILL.md) में रहता है।

## रोडमैप (Roadmap)

- **Start** (शुरुआत) — `start-package` (ढांचा बनाना), `start-astro` (Astro बनाएं) और `skrapi` (मौजूदा कोडबेस का अध्ययन करना) आज उपलब्ध हैं; और अधिक `start-*` ढांचा बनाने वाले आने वाले हैं।
- **Middle** (मध्य) — लक्षित, ऑन-डिमांड सुधारक (प्रदर्शन, सुरक्षा, UX, डेड-कोड हटाना) योजनाबद्ध हैं।
- **End** (अंत) — `end` आज उपलब्ध है; और अधिक रनटाइम, अतिरिक्त विशेष रिफैक्टरींग मोड और यूटिलिटीज (utilities) की योजना है।

प्रत्येक स्किल अपने स्वयं के समर्पित दस्तावेज़ीकरण के साथ आती है (जैसे वर्तमान [End – प्रोजेक्ट को रिफैक्टरींग करें](../../skills/end/README.md))।
