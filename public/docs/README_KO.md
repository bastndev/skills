## 세 가지 단계

| 단계 | 목적 | 주요 기능 | 예시 스킬 | 상태 |
| --- | --- | --- | --- | --- |
| **Start** | 프로젝트 초기화 | 프로덕션 준비 폴더 구조 생성, 프레임워크 초기화, 첫 커밋부터 모범 사례 적용. | `start-nextjs`... | 계획됨 |
| **Middle** | 지속적인 개선 | UI/UX 향상, 보안 강화, 성능 향상, 로직 정리, 데드 코드 제거. | TBD | 계획됨 |
| **End** | 감사, 진단 및 리팩터링 | 전체 아키텍처 및 품질 분석. **명시적인 승인 하에** 실행되는 우선순위가 지정된 단계별 계획. | `refactor-project` | ✅ |

## 사용 가능한 스킬

| 스킬 | 설명 |
| --- | --- |
| **[end](../../skills/end)** | **`refactor-project`** — 프로젝트의 처음부터 끝까지 이해합니다. 구체적인 참조와 함께 명확한 진단을 제공합니다. 올바른 아키텍처 방향을 권장하고 질서 있는 실행 계획을 구축합니다. 모든 권한은 여러분에게 있습니다. |

## 설치

```bash
# 현재 스킬 추가 (End / refactor-project)
npx skills add bastndev/skills --skill end
```

향후 스킬도 동일한 방식으로 설치할 수 있습니다:

```bash
npx skills add bastndev/skills --skill start-nextjs
```

## End 스킬의 작동 방식

1. **분석만** — 진입점을 매핑하고 구조를 이해합니다. **변경하지 않습니다.**
2. **구조화된 보고서** — 발견 사항은 버그, 위험, 기회 및 기술 부채로 분류됩니다.
3. **맞춤형 계획** — 단계는 프로젝트의 실제 가치에 따라 정렬됩니다.
4. **단계별 실행** — 승인 후 _하나의_ 단계를 실행하고 대기합니다.
5. **안전 제일** — 테스트가 없으면 생성하지 않으며 허가 없이 종속성을 추가하지 않습니다.

[skills/end/SKILL.md](../../skills/end/SKILL.md) 에서 전체 사양을 확인하세요.
