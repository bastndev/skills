## 3つのフェーズ

| フェーズ | 目的 | 主要機能 | スキル例 | ステータス |
| --- | --- | --- | --- | --- |
| **Start** | プロジェクトの初期化 | 本番環境に対応したフォルダ構造の作成、フレームワークの初期化、最初のコミットからベストプラクティスを適用。 | `start-nextjs`... | 計画中 |
| **Middle** | 継続的な改善と洗練 | UI/UXの向上、セキュリティの強化、パフォーマンス向上、ロジックのクリーンアップ。 | TBD | 計画中 |
| **End** | 監査、診断、リファクタリング | 完全なアーキテクチャと品質分析。**明示的な承認がある場合のみ**実行される優先順位付きのフェーズ別プラン。 | `refactor-project` | ✅ |

## 利用可能なスキル

| スキル | 説明 |
| --- | --- |
| **[end](../../skills/end)** | **`refactor-project`** — プロジェクト全体を理解し、具体的な参照を伴う明確な診断を提供します。適切なアーキテクチャの方向性を推奨し、整然とした実行計画を構築します。常にあなたが完全にコントロールします。 |

## インストール

```bash
# 現在のスキルを追加 (End / refactor-project)
npx skills add bastndev/skills --skill end
```

今後のスキルも同じ方法でインストール可能になります:

```bash
npx skills add bastndev/skills --skill start-nextjs
```

## Endスキルの仕組み

1. **分析のみ** — エントリポイントをマッピングし、構造を理解します。**変更は一切行いません。**
2. **構造化レポート** — 発見事項はバグ、リスク、機会、技術的負債に分類されます。
3. **カスタマイズされた計画** — フェーズはプロジェクトにとっての真の価値順に並べられます。
4. **ステップバイステップの実行** — 承認後、_1つの_フェーズを実行して停止します。
5. **安全性第一** — テストが存在しない場合は決して作成せず、許可なく依存関係を追加しません。

完全な仕様は [skills/end/SKILL.md](../../skills/end/SKILL.md) をご覧ください。

## ロードマップ

- **Start** ファミリー: 人気のスタック向けのワンコマンドプロジェクトジェネレーター。
- **Middle** ファミリー: 開発中に呼び出す専門的な改善スペシャリスト。
- **End** ファミリー: 機能の拡張と追加言語のサポート。
