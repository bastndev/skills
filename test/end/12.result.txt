  🏗️ Architecture
  
  Decision 2️⃣:
  Small architecture adjustments needed.
  
  Why: main.ts has grown into a god class with too many responsibilities
  — splitting
  message handlers into a dedicated host-message-router and extracting
  the HTML
  builder into webview-builder would reduce its size by ~60% without
  touching
  the overall two-phase webview design.
  
  Before:
  
  src/clihub/
  ├── main.ts                    # 560 lines, 10+ responsibilities
  └── index.ts                   # 680 lines, data + UI mixed
  
  After:
  
  src/clihub/
  ├── main.ts                    # provider lifecycle only (~120 lines)
  ├── host-message-router.ts     # extracted from main.ts (message
  dispatch)
  ├── webview-builder.ts         # extracted from main.ts (HTML
  assembly)
  └── index.ts                   # unchanged (browser bundle boundary)
  
  ──────────────────────────────────────────────────────────────────────
  
  🗺️ Proposed Plan
  
  Phase 1 — Extract message router from CliHubViewProvider
  
  Goal: Pull all onDidReceiveMessage handler logic out of main.ts into a
        dedicated host-message-router.ts, reducing main.ts to lifecycle
  only.
  
  Affected files: 2
  - `src/clihub/main.ts`
  - `src/clihub/host-message-router.ts`   (new)
  
  Why now: This is the highest-leverage change — it unblocks everything
  else by
           making main.ts readable before touching HTML assembly or
  agent data.
  
  Validation: check-types + lint
  
  
  Phase 2 — Extract HTML/webview builder
  
  Goal: Move _getHtmlForWebview, _getAgentHtmlForWebview, and all
  helpers
        (_getCliHubAssetUri, _getWebviewUri, _getNonce,
  serializeJsonForHtmlScript)
        into webview-builder.ts, leaving CliHubViewProvider with only
  lifecycle
        and routing concerns.
  
  Affected files: 2
  - `src/clihub/main.ts`
  - `src/clihub/webview-builder.ts`   (new)
  
  Why now: After Phase 1 main.ts is already thinner; this finishes the
  split
           and makes the HTML-assembly logic independently testable.
  
  Validation: check-types + lint
  
  
  Phase 3 — Deduplicate getAgentSlug
  
  Goal: Move getAgentSlug to a shared module and import it in both
  index.ts
        and terminal.ts, eliminating the duplicated if-chain.
  
  Affected files: 3
  - `src/clihub/webview/shared/agent-slug.ts`   (new)
  - `src/clihub/index.ts`
  - `src/clihub/webview/ui/panel-terminal/terminal.ts`
  
  Why now: Quick, self-contained, zero risk — the duplicate will
  otherwise drift
           as new agents are added.
  
  Validation: check-types + lint
  
  
  Phase 4 — Fix silent file-listing failure in _handleWorkspaceListFiles
  
  Goal: Replace the silent empty-array fallback with a webview error
  message so
        the @-mention picker shows a visible failure instead of an empty
  list.
  
  Affected files: 1
  - `src/clihub/main.ts`
  
  Why now: User-facing silent failure is worse than a visible error;
  this is
           a contained one-method fix with no architectural impact.
  
  Validation: check-types + manual verification of the @-mention picker
