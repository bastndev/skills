  🔍 Project Understanding
  
  F1 is a VS Code extension that embeds a multi-session CLI launcher
  panel. The entry point (src/extension.ts) registers a single
  WebviewViewProvider (CliHubViewProvider). Two HTML phases live inside
  that single panel view: a fuzzy-search launcher (index.ts + index.html)
  that lets users pick a CLI, and a full terminal UI (terminal.ts +
  xterm.js) once an agent is chosen. PTY processes are isolated in a
  subprocess (pty-host.ts) spawned per session. All communication crosses
  two message buses: webview ↔ host (postMessage) and host ↔ pty-host
  (Node IPC). Host-side tools (translation, TTS, spell-check, image prep)
  live in tools-cli-core/. The project compiles four esbuild targets and
  has no test suite.
