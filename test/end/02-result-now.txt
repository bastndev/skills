  🟠 Risks
  
  -
  src/clihub/webview/core/terminal-cli/session-manager.ts:getPtyHostPat
  h
  
    Problem: Uses __dirname to build path to pty-host.js — __dirname in
  the extension host points to dist/, making the resolved path
  dist/clihub/webview/core/terminal-cli/pty-host.js, but the
  double-clihub segment could break if the output layout changes.
    Impact: Silent failure at runtime; CLI sessions never start with an
  opaque "Could not start PTY host" error.
    Recommendation: Drive the path from extensionUri passed down from
  CliHubViewProvider, as is already done for all other asset paths.
  
  - src/clihub/main.ts:_handleWorkspaceListFiles
  
    Problem: vscode.workspace.findFiles is capped at 1000 results but
  silently truncates larger workspaces; the caller has no way to know
  the list is incomplete.
    Impact: @-file mention picker in the prompt modal shows an
  incomplete file tree in large repos without any indication.
    Recommendation: Either expose a truncated: boolean flag in the
  response or increase / paginate the limit.
  
  - src/clihub/main.ts:_handlePromptPrepare
  
    Problem: Falls back to { globalStorageUri: undefined } as any when
  _extensionContext is absent, which is only possible in tests; the
  preparePromptForCLI function uses globalStorageUri to write image
  files and will throw if it is undefined and attachments are present.
    Impact: Image attachments silently fail or throw an unhandled error
  when context is missing.
    Recommendation: Guard against !this._extensionContext before
  proceeding, or enforce _extensionContext as a required constructor
  argument.

  🟢 Refactoring Opportunities
  
  - src/clihub/index.ts:getAgentSlug +
  src/clihub/webview/ui/panel-terminal/terminal.ts:getAgentSlug
  
    Problem: Identical getAgentSlug function is copy-pasted in both
  files — index.ts returns '' for unknown agents, terminal.ts returns
  'default'; the logic itself is otherwise line-for-line the same.
    Impact: Any new agent requires updating two files; the diverging
  fallback return values are a latent inconsistency.
    Recommendation: Extract to src/clihub/webview/shared/agent-slug.ts
   and import from both call sites.
  
  - src/clihub/main.ts (class CliHubViewProvider)
  
    Problem: The class is 460+ lines handling webview lifecycle, 7
  distinct message types, HTML construction for two phases, workspace
  utilities, voice, translation, and spellcheck — all in one file.
    Impact: High cognitive load; any new feature or bug fix requires
  navigating the entire class.
    Recommendation: Extract each handler group (prompt.*, voice.*,
  workspace.*, HTML builders) into dedicated private modules, leaving
  CliHubViewProvider as a thin coordinator.
  
  - src/clihub/main.ts:launcherAgents +
  src/clihub/webview/core/terminal-cli/agents.ts:cliAgents
  
    Problem: Each agent requires two parallel registration entries
  (display metadata in main.ts, command metadata in agents.ts),
  connected only by matching label strings.
    Impact: Adding an agent requires touching two files and trusting
  string matching; a label typo produces no compile-time error.
    Recommendation: Merge into a single agent registry in agents.ts
  with all fields, and derive both the launcher list and the runtime
  CLI list from it.
  
  - src/clihub/main.ts:_getAgentLaunchExtensionMode
  
    Problem: Switch over vscode.ExtensionMode using three explicit if
   branches where an inline map would be cleaner and exhaustive.
    Impact: Minor readability cost; low priority.
    Recommendation: Replace with a simple Map<vscode.ExtensionMode,
  AgentLaunchExtensionMode> lookup.

  🟡 Technical Debt
  
  - src/clihub/main.ts + src/clihub/index.ts
  
    Problem: launcherAgents (display + icons) and cliAgents (commands)
  are maintained as separate arrays that must be kept in sync manually;
  there is no type-level guarantee they share the same set of agents.
    Impact: Future agent additions are error-prone and the relationship
  between the two arrays is implicit.
    Recommendation: Consolidate into a unified agent definition in
  agents.ts (see Opportunity #3).
  
  - src/clihub/webview/core/terminal-cli/installation.ts:commandExists
  
    Problem: Uses shell injection via string interpolation (command -v
  -- '${command.replace(...)}') when spawning sh -lc; the single-quote
  escaping is correct but the pattern is fragile and platform-specific.
    Impact: Any future change to escaping risks shell injection;
  approach diverges by platform already.
    Recommendation: Prefer which/where with array args and no shell,
  consistent with the win32 branch already using where.exe with array
  args.
