# Skills — astro-test

## Skills (.agents/skills/)

### 1. **auth0-quickstart**
Guía para integrar Auth0 en aplicaciones web de forma rápida. Proporciona instrucciones paso a paso para implementar autenticación con OAuth 2.0 / OpenID Connect.
**Triggers**: cuando el usuario necesita añadir login, registro, o gestión de usuarios con Auth0.

### 2. **gpt-image-2**
Skill para generar imágenes usando DALL-E 2 / 3 a través de la API de OpenAI. Incluye flujo completo desde la generación hasta la descarga de imágenes.
**Triggers**: cuando se necesita crear, editar o variar imágenes mediante prompts de texto.

### 3. **improve-codebase-architecture**
Metodología estructurada para analizar y mejorar la arquitectura de código existente. Incluye múltiples documentos de referencia (DEEPENING.md, INTERFACE-DESIGN.md, HTML-REPORT.md, LANGUAGE.md) para guiar refactorizaciones y mejoras estructurales.
**Triggers**: cuando se pide revisar, refactorizar o mejorar la organización del código, especialmente en proyectos legacy o con deuda técnica.

### 4. **skrapi**
Analiza la arquitectura de proyectos de software (web, mobile, extensiones, librerías) en cualquier stack JS/TS (Next.js, Astro, React Native, etc.) y genera documentación enfocada en Markdown (ARCHITECTURE.md, PACKAGES.md, PROMPT.md, opcionalmente SKILLS.md y otros).
**Triggers**: cuando el usuario quiere entender o documentar un codebase existente, estudiar otro proyecto, o saber qué package maneja una funcionalidad específica.

## Skills (.claude/skills/)

### 1. **bun**
Documentación y mejores prácticas para trabajar con Bun como runtime y package manager. Cubre comandos, configuración, y particularidades vs npm/yarn/pnpm.
**Triggers**: cuando el proyecto usa `bun.lock`, se ejecutan scripts con `bun`, o se necesita ayuda específica con Bun.

### 2. **vercel-react-best-practices**
Guías de optimización de rendimiento para React y Next.js desde Vercel Engineering (108KB de contenido). Incluye patrones de data fetching, bundle optimization, y mejores prácticas de renderizado.
**Triggers**: cuando se trabaja con React/Next.js y se necesita mejorar performance, optimizar bundles, o revisar patrones de código.
