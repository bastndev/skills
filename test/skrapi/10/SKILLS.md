# Skills — astro-test

## Skills (.agents/skills/)

### 1. **auth0-quickstart**
Integración rápida de Auth0 para aplicaciones web. Proporciona guías de configuración y snippets de código para implementar autenticación y autorización con Auth0.  
**Triggers**: Cuando se necesita agregar login, signup, o gestión de usuarios con Auth0 al proyecto.

### 2. **bun**
Documentación y helpers para usar Bun como runtime y package manager. Incluye comandos específicos de Bun, optimizaciones, y diferencias con npm/yarn/pnpm.  
**Triggers**: Cuando se trabaja con scripts, instalación de paquetes, o configuración relacionada con Bun.

### 3. **gpt-image-2**
Generación de imágenes con GPT-image-2. Skill especializado en crear imágenes vía prompts, útil para prototipos rápidos o assets placeholder.  
**Triggers**: Cuando se solicita generar imágenes, crear assets visuales, o mockups automáticos.

### 4. **improve-codebase-architecture**
Análisis y mejora de arquitectura de código. Revisa patrones, separación de responsabilidades, y sugiere refactorizaciones estructurales. Incluye archivos de referencia para diseño de interfaces, profundización de análisis, reportes HTML, y guías de lenguaje.  
**Triggers**: Cuando se pide revisar la arquitectura, mejorar la organización del código, o identificar code smells arquitectónicos.

### 5. **skrapi**
Analiza la arquitectura de proyectos de software y genera documentación estructurada en formato Markdown. Detecta el tipo de proyecto (web/mobile/extension/library), examina stack, dependencias, routing, estado, y patrones. Produce archivos `ARCHITECTURE.md`, `PACKAGES.md`, `SKILLS.md` (si aplica), y `PROMPT.md` con comandos de instalación reutilizables. Incluye archivos de referencia específicos por tipo de proyecto (`web.md`, `mobile.md`, `extension.md`, `library.md`).  
**Triggers**: Cuando se solicita analizar o documentar la arquitectura del proyecto, entender la estructura del código, o generar documentación técnica automática.

### 6. **vercel-react-best-practices**
Guías de optimización de rendimiento para React y Next.js desde Vercel Engineering. Cubre patrones de data fetching, optimización de bundle, Server Components, hidratación selectiva, y performance best practices.  
**Triggers**: Cuando se escribe, revisa o refactoriza código React/Next.js con foco en rendimiento y optimizaciones.
